# คู่มือการติดตั้งและจัดตั้งระบบบน Raspberry Pi 4 อย่างละเอียด (Production Deployment)
### ระบบควบคุมการเข้าออกห้อง คณะครุศาสตร์ มทร.พระนคร (SmartAccess)

คู่มือนี้จะอธิบายขั้นตอนการนำระบบเว็บ **Next.js + postgreSQL** ไปติดตั้งบนบอร์ดคอมพิวเตอร์ขนาดเล็ก **Raspberry Pi 4** เพื่อจัดตั้งเป็น **Local Server** ประจำตึกเรียน/ห้องเรียน ให้ทำงานร่วมกับกล่องควบคุม **ESP32** แบบออฟไลน์ภายในวงแลนเดียวกันได้อย่างเสถียรที่สุด 24 ชั่วโมงต่อวัน โดยไม่ต้องพึ่งพาระบบคลาวด์ภายนอก

---

## 📋 อุปกรณ์ที่ต้องจัดเตรียม
1. **บอร์ดคอมพิวเตอร์**: Raspberry Pi 4 (แนะนำรุ่น RAM 4GB หรือ 8GB เพื่อการบิลด์ Next.js ที่รวดเร็ว)
2. **สื่อบันทึกข้อมูล**: MicroSD Card ความจุขั้นต่ำ 16GB (แนะนำ Class 10 หรือความเร็วระดับ A1/A2 ขึ้นไปเพื่อประสิทธิภาพที่ดี)
3. **แหล่งจ่ายไฟ**: อะแดปเตอร์จ่ายไฟ Type-C (5V 3A ของ Raspberry Pi แท้เพื่อป้องกันไฟไม่พอ)
4. **เครื่องอ่านการ์ด**: MicroSD Card Reader เพื่อเชื่อมต่อกับคอมพิวเตอร์หลักของคุณ
5. **ระบบเน็ตเวิร์ก**: เราเตอร์ Wi-Fi (Wi-Fi Router) ที่จะนำมาประจำการเชื่อมต่อในห้องปฏิบัติการ

---

## 🛠️ ขั้นตอนที่ 1: การดาวน์โหลดและติดตั้งระบบปฏิบัติการ (Flash OS)

เราจะใช้เครื่องมือ **Raspberry Pi Imager** ในการเขียนระบบปฏิบัติการลงในการ์ด ซึ่งง่ายและสามารถกำหนดค่าล่วงหน้าได้ทันที:

1. ดาวน์โหลดโปรแกรม [Raspberry Pi Imager](https://www.raspberrypi.com/software/) และติดตั้งบนคอมพิวเตอร์หลักของคุณ
2. เสียบ MicroSD Card เข้ากับคอมพิวเตอร์หลักผ่าน Card Reader
3. เปิดโปรแกรม Raspberry Pi Imager:
   * **Choose Device**: เลือก **Raspberry Pi 4**
   * **Choose OS**: เลือก **Raspberry Pi OS (64-bit) Lite** (รุ่น Lite ไม่มีหน้าจอ UI ซึ่งประหยัด RAM และกินไฟต่ำมาก เหมาะสำหรับการรันเป็นเซิร์ฟเวอร์เว็บโดยเฉพาะ)
   * **Choose Storage**: เลือก MicroSD Card ของคุณ
4. กด **Next** โปรแกรมจะถามว่าต้องการแก้ไขการตั้งค่าระบบล่วงหน้าหรือไม่ (OS Customisation) ให้เลือก **EDIT SETTINGS**:
   * **General**:
     - ตั้งค่า Username เป็น `pi` และ Password ตามที่คุณกำหนด (เช่น `pi1234`)
     - ตั้งค่า Wi-Fi: กรอกชื่อ SSID และรหัสผ่านของเครือข่าย Wi-Fi ที่ติดตั้งภายในห้องเรียน
     - เลือกโซนเวลา (Timezone) เป็น **Asia/Bangkok** และแป้นพิมพ์เป็น **us**
   * **Services**:
     - ทำเครื่องหมายที่ **Enable SSH** และเลือก **Use password authentication** เพื่อเปิดให้เราล็อกอินเข้าไปตั้งค่าระยะไกลได้
5. กด **Save** แล้วกด **Yes** เพื่อเริ่มเขียนข้อมูลลงการ์ด (ข้อมูลเก่าในการ์ดจะถูกลบทั้งหมด)

---

## 🛠️ ขั้นตอนที่ 2: การเปิดเครื่องและเชื่อมต่อเข้าเซิร์ฟเวอร์ (Boot & SSH)

1. เมื่อเขียนระบบปฏิบัติการเสร็จแล้ว ให้ดึง MicroSD Card มาเสียบเข้าที่บอร์ด Raspberry Pi 4
2. ทำการเสียบปลั๊กอะแดปเตอร์เพื่อจ่ายไฟเข้าบอร์ด Raspberry Pi 4 (บอร์ดจะเปิดเครื่องและเชื่อมต่อเข้ากับ Wi-Fi ที่ตั้งค่าล่วงหน้าโดยอัตโนมัติภายใน 1 นาที)
3. เปิดเครื่องคอมพิวเตอร์หลักของคุณ (ที่ต่ออยู่ในวง Wi-Fi เดียวกัน) จากนั้นเปิดโปรแกรม **Terminal (macOS/Linux)** หรือ **Command Prompt / PowerShell (Windows)**
4. ทำการล็อกอินเข้าไปในบอร์ดผ่านระบบ SSH:
   ```bash
   ssh pi@raspberrypi.local
   ```
   *(หมายเหตุ: หากคุณเปลี่ยนชื่อโฮสต์ตอนฟลัช ให้พิมพ์ `ssh pi@ชื่อโฮสต์ของคุณ.local` หรือใช้โปรแกรมสแกนหา IP ของ Raspberry Pi จากหน้าตั้งค่าเราเตอร์แล้วสั่ง `ssh pi@192.168.1.XX` แทน)*
5. ระบบจะถามหารหัสผ่าน ให้กรอกรหัสผ่านที่คุณตั้งไว้ (ระบบจะไม่แสดงตัวหนังสือขณะพิมพ์เพื่อความปลอดภัย ให้พิมพ์รหัสผ่านแล้วกด Enter ได้เลย)
6. เมื่อล็อกอินสำเร็จ ให้รันคำสั่งอัปเดตระบบปฏิบัติการให้เป็นเวอร์ชันล่าสุด:
   ```bash
   sudo apt update && sudo apt upgrade -y
   ```

---

## 🛠️ ขั้นตอนที่ 3: การติดตั้งสภาพแวดล้อมระบบเว็บ (Install Node.js & NPM)

ระบบเว็บหลังบ้าน Next.js ต้องการสภาพแวดล้อมรันไทม์ Node.js ในการทำงาน:

1. สั่งดึงคลังดาวน์โหลดโปรแกรมเวอร์ชันเสถียร Node.js 20.x (LTS):
   ```bash
   curl -fsSL https://deb.nodesource.com/setup_20.x | sudo -E bash -
   ```
2. ทำการสั่งติดตั้ง Node.js และแพ็กเกจ NPM:
   ```bash
   sudo apt install -y nodejs
   ```
3. ตรวจสอบการติดตั้งว่าสำเร็จและเวอร์ชันถูกต้องหรือไม่:
   ```bash
   node -v
   npm -v
   ```

---

## 🛠️ ขั้นตอนที่ 4: การติดตั้งและจัดตั้งฐานข้อมูล (Install postgreSQL)

เราจะใช้ **postgreSQL** ซึ่งเป็นฐานข้อมูลที่ทำงานได้รวดเร็ว ปลอดภัย และยืดหยุ่นสูง:

1. สั่งติดตั้ง postgreSQL Server:
   ```bash
   sudo apt install -y postgresql postgresql-contrib
   ```
2. รันคำสั่งเข้าไปตั้งค่ารหัสผ่านสำหรับผู้ใช้ `postgres`:
   ```bash
   sudo -u postgres psql
   ```
   และพิมพ์คำสั่งเปลี่ยนรหัสผ่านเพื่อความปลอดภัย:
   ```sql
   ALTER USER postgres PASSWORD 'รหัสผ่านฐานข้อมูลที่คุณต้องการ';
   ```
3. สร้าง Database หลักของระบบ:
   ```sql
   CREATE DATABASE smartaccess_access;
   \q
   ```

---

## 🛠️ ขั้นตอนที่ 5: การนำซอร์สโค้ดระบบเว็บขึ้นเซิร์ฟเวอร์ (Deploy Code)

1. สั่งติดตั้งเครื่องมือ Git ในกรณีที่ยังไม่มีในบอร์ด:
   ```bash
   sudo apt install -y git
   ```
2. สั่งโคลนโปรเจกต์ของคุณมาจาก GitHub (หรือจะใช้วิธีโยนไฟล์ผ่านโปรแกรม FTP เช่น FileZilla ก็ได้):
   ```bash
   # โคลนโปรเจกต์มาวางไว้ที่ไดเรกทอรีบ้านของ Pi
   git clone https://github.com/your-username/your-repo.git
   cd your-repo/my-app
   ```
3. สั่งติดตั้งไลบรารีพัฒนาซอฟต์แวร์ทั้งหมดของตัวเว็บ:
   ```bash
   npm install
   ```

---

## 🛠️ ขั้นตอนที่ 6: การเขียนค่าสิ่งแวดล้อมลับเฉพาะเครือข่ายจริง (.env.local)

1. สร้างหรือเปิดไฟล์ `.env.local` เพื่อระบุการเชื่อมต่อภายในเครือข่ายจริง:
   ```bash
   nano .env.local
   ```
2. คัดลอกข้อความด้านล่างนี้ลงไปในไฟล์ โดยแก้ไขรหัสผ่านและ IP ให้ตรงกับความเป็นจริง:
   ```env
   # การเชื่อมโยงฐานข้อมูล postgreSQL (ที่รันอยู่ในตัวเครื่อง Pi เอง)
   POSTGRES_HOST=localhost
   POSTGRES_PORT=5432
   POSTGRES_USER=postgres
   POSTGRES_PASSWORD=ใส่รหัสผ่านฐานข้อมูลที่คุณตั้งในขั้นตอนที่ 4
   POSTGRES_DATABASE=smartaccess_access

   # รหัสความปลอดภัย JWT
   JWT_SECRET=smartaccess-door-access-super-secret-key-2024

   # การเชื่อมโยงกล่องควบคุม ESP32
   ESP32_IP=ระบุไอพีของบอร์ด ESP32 จริง (เช่น 192.168.1.100)
   ESP32_PORT=80
   ESP32_MOCK_MODE=false

   # ช่องแจ้งเตือนกลุ่มแชท Discord
   DISCORD_WEBHOOK_URL=ลิงก์ Webhook ของดิสคอร์ดคุณ

   # ระบุ URL ของเครื่องราสเบอร์รี่พายเพื่อใช้เจน QR code
   NEXT_PUBLIC_APP_URL=http://ระบุไอพีของเครื่องราสเบอร์รี่พาย:3000
   ```
3. กด **`Ctrl + O`** แล้วกด **`Enter`** เพื่อเซฟไฟล์ จากนั้นกด **`Ctrl + X`** เพื่อออกจากหน้าต่างแก้ไข

---

## 🛠️ ขั้นตอนที่ 7: การคอมไพล์ระบบและสร้างตารางฐานข้อมูลอัตโนมัติ

1. รันคำสั่งบิลด์ Next.js ให้เป็นรุ่น Production ที่ทำงานได้เสถียรและเร็วสูงสุด:
   ```bash
   npm run build
   ```
2. เมื่อบิลด์สำเร็จ ระบบจะตรวจสอบตารางฐานข้อมูลและเริ่มเชื่อมโยงกับ postgreSQL อัตโนมัติในครั้งแรกที่ถูกเรียกใช้งาน เพื่อรันสคริปต์ Migration ตารางทั้งหมดทันทีโดยที่คุณไม่ต้องไปสั่งสร้างตารางใน SQL เอง

## 🛠️ ขั้นตอนที่ 8: การเปิดระบบ 24 ชั่วโมงด้วยตัวคุม PM2 (Daemon Service)

หากรันด้วยคำสั่งปกติ เมื่อเราปิดหน้าต่าง SSH ไป ตัวเว็บจะหยุดทำงาน เราจึงต้องติดตั้ง **PM2** เพื่อทำหน้าที่เฝ้าระวังและเปิดโปรแกรมค้างไว้ตลอดเวลา แม้ไฟดับชั่วคราวแล้วไฟกลับมา บอร์ด Pi จะเปิดเว็บนี้กลับมาเองโดยอัตโนมัติ:

1. ติดตั้งตัวจัดการโพรเซส PM2 ทั่วทั้งระบบ:
   ```bash
   sudo npm install -g pm2
   ```
2. สั่งเริ่มการทำงานตัวเว็บ Next.js ภายใต้การควบคุมของ PM2:
   ```bash
   pm2 start npm --name "smartaccess-app" -- run start
   ```
3. ตั้งค่าระบบควบคุมให้ PM2 เริ่มเปิดเว็บอัตโนมัติตอน Raspberry Pi เปิดเครื่อง (Autostart on Boot):
   ```bash
   pm2 startup
   ```
   *(หมายเหตุ: คำสั่งด้านบนจะให้ข้อความบรรทัดยาวๆ ขึ้นมาในหน้าจอ ให้ก๊อบปี้คำสั่งบรรทัดนั้นที่ขึ้นคำว่า `sudo env PATH=...` มาวางรันอีกรอบแล้วกด Enter เพื่อให้ระบบเซ็ตอัปสิทธิ์เข้าถึงลินุกซ์ได้สมบูรณ์)*
4. บันทึกรายชื่อกระบวนการที่เปิดอยู่ในปัจจุบันลงในหน่วยความจำถาวร:
   ```bash
   pm2 save
   ```

## 📡 ขั้นตอนที่ 9: ตรวจสอบความเสถียรและความรวดเร็ว
* ตอนนี้ระบบเซิร์ฟเวอร์หลัก Next.js รันอยู่บนพอร์ต `3000` ของเครื่อง Raspberry Pi 4 แล้ว
* เครื่องคอมพิวเตอร์หลักหรือโทรศัพท์มือถือในห้องปฏิบัติการ สามารถเปิดเข้ามาใช้บริการเว็บบอร์ดควบคุมแอดมินหรือสมัครใช้งานได้ทันทีผ่านลิงก์:
  👉 **`http://ไอพีของเครื่องราสเบอร์รี่พาย:3000`**
* ระบบการสื่อสาร สั่งงานอนุมัติ ข้อมูลประวัติการลบตามความถูกต้องทางกฎหมาย พ.ร.บ. คอมพิวเตอร์ และการสั่งปลดล็อกกลอน ESP32 จะทำงานตอบสนองได้อย่างคึกคัก รวดเร็ว (ดีเลย์ต่ำกว่า 5 มิลลิวินาที) และทนทานต่อการขัดข้องทางเครือข่ายอินเทอร์เน็ตภายนอกอย่างสมบูรณ์แบบครับ!
