# คู่มือการเลือกซื้อและตั้งค่าเราเตอร์ Wi-Fi สำหรับระบบ IoT ระดับองค์กร (Wi-Fi Router Guide)
### โครงการระบบควบคุมการเข้าออกห้อง คณะครุศาสตร์ มทร.พระนคร (SmartAccess)

เราเตอร์ Wi-Fi เปรียบเสมือน **"หัวใจและเส้นเลือดใหญ่"** ของระบบควบคุมประตูนี้ เนื่องจากบอร์ด **ESP32** และเซิร์ฟเวอร์ **Raspberry Pi 4** จะต้องสื่อสารกันแบบไร้สายตลอด 24 ชั่วโมง การเลือกซื้อเราเตอร์ที่เหมาะสมและตั้งค่าอย่างถูกต้องจะส่งผลต่อความเร็ว (Latency) และความเสถียรในการเปิดประตูโดยตรง (ป้องกันบอร์ดค้าง/ออฟไลน์)

---

## 🔍 ส่วนที่ 1: คุณสมบัติทางเทคนิคที่จำเป็นที่สุด (Must-Have Technical Specs)

ในการทำระบบ IoT สำหรับบอร์ดไมโครคอนโทรลเลอร์อย่าง **ESP32** ห้ามใช้เราเตอร์แถมทั่วไปจากผู้ให้บริการอินเทอร์เน็ตเด็ดขาด เนื่องจากไม่เสถียรและมักจำกัดจำนวนอุปกรณ์ โดยเราเตอร์ที่เลือกซื้อต้องมีสเปกดังนี้:

### 1. รองรับย่านความถี่ 2.4 GHz ที่มีเสถียรภาพสูง
* **ข้อเท็จจริง**: บอร์ด **ESP32 รองรับเฉพาะ Wi-Fi คลื่นย่าน 2.4 GHz (802.11 b/g/n) เท่านั้น** (ไม่รองรับ 5 GHz)
* **เกณฑ์การเลือก**: ต้องมองหาเราเตอร์ที่มีกำลังส่งสัญญาณย่าน 2.4 GHz ที่นิ่ง มีเทคโนโลยีจัดคิวสัญญาณที่ดี แม้จะมีสัญญาณรบกวนในตึกมหาวิทยาลัยหนาแน่น

### 2. เทคโนโลยี Wi-Fi 6 (802.11ax)
* **เหตุผล**: Wi-Fi 6 มีเทคโนโลยี **OFDMA** และ **MU-MIMO** ที่ช่วยให้เราเตอร์คุยกับอุปกรณ์จำนวนมากได้พร้อมกันโดยไม่มีอาการหน่วง และประหยัดพลังงานของอุปกรณ์ IoT รวมถึงจัดการสัญญาณชนกัน (Collision) ภายในตึกเรียนได้ดีกว่ารุ่นเก่า (Wi-Fi 5) มาก

### 3. ฟังก์ชันจองเลขไอพีถาวร (DHCP IP Reservation / Static IP Binding)
* **ความสำคัญ**: เราเตอร์ต้องมีระบบจอง IP จากหน้าเฟิร์มแวร์ที่ใช้งานง่าย โดยระบุ MAC Address ของ ESP32 และ Raspberry Pi เพื่อแจก IP เดิมให้ทุกครั้งที่บอร์ดเปิดเครื่อง ป้องกันไม่ให้ IP สลับเครื่องซึ่งจะทำให้ส่งคำสั่งเปิดประตูไม่ได้

### 4. ตัวเลือกเปิด-ปิดฟังก์ชัน AP Isolation (Client Isolation)
* **สำคัญมาก**: เราเตอร์ระดับองค์กรหรือระบบ Wi-Fi ของมหาวิทยาลัยส่วนใหญ่จะเปิดฟังก์ชัน **AP Isolation** ไว้ (ห้ามไม่ให้อุปกรณ์เชื่อมต่อ Wi-Fi คุยกันเองเพื่อความปลอดภัย)
* **เกณฑ์การเลือก**: เราเตอร์ที่เราซื้อมาจัดตั้งเองในห้อง **จะต้องรองรับการ "ปิด" (Disable) AP Isolation** เพื่อให้เซิร์ฟเวอร์ Next.js สามารถส่งคำสั่ง HTTP ยิงปลดล็อกไปยังเครื่อง ESP32 ได้โดยตรง

## 🏷️ ส่วนที่ 2: รุ่นเราเตอร์ที่แนะนำ (Recommended Brands & Models)

จากการทดสอบความเสถียรสำหรับงานระบบเครือข่ายภายในเฉพาะห้องปฏิบัติการ IoT รุ่นที่แนะนำมีดังนี้ครับ:

### 1. กลุ่มราคาประหยัดสำหรับ 1-2 ห้อง (Budget / Classroom Lab)
*ราคาประมาณ 1,000 - 1,800 บาท*
เหมาะสำหรับการควบคุมกล่องประตู 1-2 บอร์ด และแอดมินใช้งานทั่วไปภายในห้องปฏิบัติการ
* **TP-Link Archer AX12 / AX23 (Wi-Fi 6)**: 
  - *ข้อดี*: ราคาเข้าถึงง่ายที่สุด ตัวชิปเซ็ตประมวลผลดี เฟิร์มแวร์ภาษาไทย ใช้ง่าย มีเมนูจอง IP (Address Reservation) ที่ไม่ซับซ้อน ปิด AP Isolation ง่าย
* **ASUS RT-AX1800HP / RT-AX53U**:
  - *ข้อดี*: เสาสัญญาณกำลังส่งสูงมาก (High Power) เสถียรสูง เหมาะกับห้องเรียนที่เป็นผนังปูนหนา

### 2. กลุ่มระดับกลาง ประสิทธิภาพสูง (Mid-Range / High-Density Classroom)
*ราคาประมาณ 2,000 - 3,500 บาท*
เหมาะสำหรับห้องแล็บคอมพิวเตอร์ขนาดใหญ่ หรือการเชื่อมโยงระบบควบคุมบอร์ดประตูหลายๆ จุดในตึกเดียวกัน
* **ASUS RT-AX55 / RT-AX57**:
  - *ข้อดี*: ตัวประมวลผล Dual-Core ความเร็วสูง มีความทนทานและความเสถียรในการส่งผ่านข้อมูล IoT สูงมาก ระบบความปลอดภัยเครือข่ายดีเยี่ยม
* **TP-Link Deco X20 (Mesh Wi-Fi)**:
  - *ข้อดี*: ในกรณีที่ต้องการเดินระบบให้ครอบคลุมหลายห้องในชั้นเรียนเดียวกัน สามารถนำมาติดตั้งพ่วงกันแบบไร้สายและเดินระบบสายแลนได้เป็นระเบียบ

## ⚙️ ส่วนที่ 3: ขั้นตอนการตั้งค่าเครือข่ายให้ทำงานแบบลื่นไหลและไร้บั๊ก

หลังจากแกะกล่องเราเตอร์ Wi-Fi ตัวใหม่แล้ว ให้ตั้งค่าตามขั้นตอนระดับมืออาชีพต่อไปนี้เพื่อให้การเชื่อมต่อของ ESP32 เสถียรและเร็วสูงสุด:

### 1. แยกชื่อ Wi-Fi (SSID) ระหว่าง 2.4 GHz และ 5 GHz
* **อย่าใช้ระบบ Smart Connect (ที่รวมชื่อไวไฟชื่อเดียวแล้วให้เราเตอร์สลับคลื่นอัตโนมัติ)** เพราะชิป ESP32 มักสับสนและหลุดการเชื่อมต่อบ่อยครั้ง
* **วิธีตั้งค่า**: ในหน้าเว็บเราเตอร์ ให้ปิด Smart Connect และตั้งชื่อให้ชัดเจนแยกจากกัน:
  - ไวไฟย่าน 2.4 GHz: ตั้งชื่อเป็น `SmartAccess-2.4G` (สำหรับบอร์ด **ESP32** ต่อเท่านั้น)
  - ไวไฟย่าน 5 GHz: ตั้งชื่อเป็น `SmartAccess-5G` (สำหรับแอดมินนำคอมพิวเตอร์หลักหรือมือถือมาต่อเพื่อควบคุม)

### 2. ปิดระบบป้องกันสัญญาณ AP Isolation (Client Isolation)
* เข้าหน้าเว็บเราเตอร์ ไปที่เมนู **Wireless / Advanced Wireless / Guest Network**
* มองหาตัวเลือก **AP Isolation** หรือ **Client Isolation** หรือ **Allow guests to access my local network**
* ทำการตั้งค่าให้เป็น **Disable** หรือ **Allow** เพื่อเปิดประตูให้อุปกรณ์ภายในวง Wi-Fi สามารถมองเห็นกันเองและรับส่งข้อมูลสั่งเปิดประตูหากันได้

### 3. ตั้งเวลาสัญญาเช่า DHCP (DHCP Lease Time) ให้ยาวขึ้น
* ตามปกติเราเตอร์จะเปลี่ยนและเช็คตั๋ว IP อุปกรณ์ทุกๆ 2-24 ชั่วโมง ซึ่งอาจทำให้บอร์ด ESP32 สะดุดขณะต่ออายุสัญญาณ
* **วิธีตั้งค่า**: ไปที่เมนู **LAN / DHCP Server** ปรับค่า **Lease Time** ให้สูงขึ้น (เช่น 2880 นาที หรือ 48 ชั่วโมงขึ้นไป) เพื่อความราบรื่นและไม่ต้องตัดการต่อบอร์ดบ่อยๆ

### 4. จองไอพีถาวรให้บอร์ด (DHCP Reservation)
* ไปที่เมนู **DHCP Server** -> **Address Reservation** หรือ **Static IP Lease**
* กดเพิ่มอุปกรณ์ (Add New):
  - นำ **MAC Address** ของเครื่อง Raspberry Pi 4 มาผูกไว้กับ IP: `192.168.1.5`
  - นำ **MAC Address** ของบอร์ด ESP32 มาผูกไว้กับ IP: `192.168.1.100` (ให้ตรงกับที่ระบุใน `.env.local`)
* ทำการกด Save และรีสตาร์เราเตอร์หนึ่งครั้ง บอร์ดทั้งสองจะเชื่อมต่อตรงกันและส่งสัญญาณปลดล็อกประตูภายใน 0.005 วินาทีตลอดกาลครับ!
